# Добавление контента

## Добавить дисциплину

Создай файл в `src/content/subjects/`. Имя файла — латиницей, через дефис, например `ray-tracing.md`.

```markdown
---
id: 27
title: "Название дисциплины"
image: "/resources/subjects/image.png"
stage: "advanced"
order: 6
description: "Описание дисциплины."
resources:
  - level: easy
    language: en
    url: "https://url.com"
    title: "Название ресурса"
    description: "Краткое описание"
  - level: medium
    title: 'Автор "Название книги"'
    description: "Краткое описание"
  - level: hard
    url: "https://url.com"
    title: "Название"
---
```

**Поля дисциплины:**
- `id` — уникальный номер (возьми следующий по порядку)
- `stage` — к какому этапу относится (см. ниже)
- `order` — порядок отображения внутри этапа
- `image` — путь к картинке из папки `public/resources/subjects/`

**Поля ресурса:**
- `level` — сложность: `easy`, `medium`, `hard` (опционально)
- `language` — язык: `ru`, `en`, `en/ru` (опционально, если не указан — считается русский)
- `url` — ссылка (опционально, если ресурс без ссылки — не добавляй поле)
- `title` — название ресурса или книги
- `description` — краткое описание (опционально)

**Совет:** если `title` содержит двойные кавычки `"` — оберни его в одинарные: `title: 'Автор "Книга"'`

## Добавить этап

Создай файл в `src/content/stages/`. Имя файла станет идентификатором этапа.

```markdown
---
stage_id: "my-new-stage"
stage_name: "Этап 5"
title: "Название этапа"
order: 8
description: "Описание этапа — зачем он нужен и что даёт."
---
```

**Поля:**
- `stage_id` — уникальный идентификатор этапа (латиницей, через дефис)
- `stage_name` — отображаемое название этапа, например `"Этап 3"`
- `order` — порядок отображения на странице
- `description` — описание этапа

Затем у дисциплин, которые относятся к этому этапу, укажи `stage: "my-new-stage"` (значение из `stage_id`).

## Добавить раздел в «Материалы»

Создай файл в `src/content/materials/`. Имя файла — латиницей, через дефис.

```markdown
---
title: "Название раздела"
order: 7
resources:
  - url: "https://url.com"
    title: "Название ресурса"
    description: "Краткое описание"
articles:
  - url: "https://url.com"
    title: "Название статьи"
    description: "Краткое описание"
---
```

**Поля:**
- `order` — порядок отображения на странице
- `resources` — список сайтов, инструментов и курсов (опционально)
- `articles` — список статей (опционально)

**Поля каждой записи:**
- `url` — ссылка
- `title` — название
- `description` — краткое описание (опционально)
