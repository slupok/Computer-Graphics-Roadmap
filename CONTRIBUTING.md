# Добавление контента

## Добавить этап

Создай файл в `src/content/stages/`. Имя файла станет идентификатором этапа.

```markdown
---
id: "my-new-stage"
stage: "Этап 5"
title: "Название этапа"
description: "Описание этапа — зачем он нужен и что даёт."
order: 8
---
```

**Поля:**
- `id` — уникальный идентификатор этапа (латиницей)
- `stage` — отображаемое название этапа, например `"Этап 3"`
- `title` — заголовок этапа
- `description` — описание этапа
- `order` — порядок отображения на странице

Затем у дисциплин, которые относятся к этому этапу, укажи `stage: "my-new-stage"` (значение из `id`).

## Добавить дисциплину

Создай файл в `src/content/subjects/`. Имя файла — латиницей, через дефис, например `ray-tracing.md`.

```markdown
---
title: "Название дисциплины"
image: "/resources/subjects/image.png"
stage: "advanced"
order: 6
description: "Описание дисциплины."
resources:
  - title: "Название ресурса"
    description: "Краткое описание"
    url: "https://url.com"
    level: easy
    language: en
  - title: 'Автор "Название книги"'
    description: "Краткое описание"
    level: medium
  - title: "Название"
    url: "https://url.com"
    level: hard
---
```

**Поля дисциплины:**
- `stage` — к какому этапу относится (см. выше)
- `order` — порядок отображения внутри этапа
- `image` — путь к картинке из папки `public/resources/subjects/`

**Поля ресурса:**
- `title` — название ресурса или книги
- `description` — краткое описание (опционально)
- `url` — ссылка (опционально)
- `level` — сложность: `easy`, `medium`, `hard` (опционально)
- `language` — язык: `ru`, `en`, `en/ru` (опционально)

**Совет:** если `title` содержит двойные кавычки `"` — оберни его в одинарные: `title: 'Автор "Книга"'`

## Добавить раздел в «Материалы»

Создай файл в `src/content/materials/`.

```markdown
---
title: "Название раздела"
order: 7
resources:
  - title: "Название ресурса"
    description: "Краткое описание"
    url: "https://url.com"
articles:
  - title: "Название статьи"
    description: "Краткое описание"
    url: "https://url.com"
---
```

**Поля:**
- `order` — порядок отображения на странице
- `resources` — список сайтов, инструментов и курсов (опционально)
- `articles` — список статей (опционально)

**Поля каждой записи:**
- `title` — название
- `description` — краткое описание (опционально)
- `url` — ссылка

## Добавить заметку

Создай файл в `src/content/notes/`. Имя файла — латиницей, через дефис, например `my-note.md`.

```markdown
---
title: "Название заметки"
date: "2026-04-14"
description: "Краткое описание заметки."
tags: ["тег1", "тег2"]
---
```

Содержимое заметки в формате **Markdown**.

## Подзаголовок

Можно использовать любые элементы Markdown: списки, ссылки, блоки кода и т.д.


**Поля:**
- `title` — название заметки
- `date` — дата в формате `YYYY-MM-DD` (опционально)
- `description` — краткое описание, отображается в списке заметок (опционально)
- `tags` — список тегов для фильтрации (опционально)
